1. Generate project structure using the template.py file
2. Perform EDA in an ipynb file in notebooks
3. Train base models to get an idea of well performing model types - track with mlflow. Add base models.png
4. perform hyperparameter tuning on best model (Gradient boosting regressor as it has decent test R2, low training time, and low test RMSE score) - track with mlflow. Add Fine tuning params.png
5. steps 2,3,4 are taken from a kaggle notebook available at https://www.kaggle.com/code/jaynandasana/91-7-seoul-bike-sharing-demand-prediction#5.-Scaling-Data-and-Model-Building-Using-Various-Regression-Techniques
6. Upload data to a mongodb cluster
7. install minikube and set up kubeflow pipelines
8. create app_secret secret for all environment variables in constants file using:-
kubectl create secret generic app_secret -n kubeflow \
  --from-literal=AWS_ENDPOINT_URL='your IP adress' \
  --from-literal=AWS_DEFAULT_REGION=us-east-1 \
  --from-literal=AWS_ACCESS_KEY_ID=test \
  --from-literal=AWS_SECRET_ACCESS_KEY=test
  ...
9. start localstack container using docker-compose, and use localstack_bootstrap/create_bucket.sh to create s3 bucket.
10. start minikube cluster and port forward for the ui.
